## 1. Built-in 함수와 메서드

```python
a = [5,7,1,2,9,10]
a.sort()
print(a)
# [1,2,5,7,9,10] 이 프린트된다. 객체 원본이 바뀐다.


a = [5,7,1,2,9,10]
sorted(a)
print(a)
# [5,7,1,2,9,10] 이 프린트된다. 객체 원본은 바뀌지 않는다. 함수 사용할 때만 바뀐다.
b = sorted(a)
print(b)
# [1,2,5,7,9,10] dl 프린트된다. 이렇게 함수로 정한 값을 다른 곳에 대응시켜야 다시 사용 가능하다.
```



## 2. .extend()와 .append()

```python
a = ['hi', 'hello', '안녕', '하세요']
a.append('반갑습니다')
print(a)
# ['hi', 'hello', '안녕', '하세요', '반갑습니다'] 이 프린트된다. 

a = ['hi', 'hello', '안녕', '하세요']
a.extend('반갑습니다')
print(a)
# ['hi', 'hello', '안녕', '하세요', '반', '갑', '습', '니', '다'] 이 프린트된다. extend 함수 괄호 안에 들어가야 하는 값이 iterable 하고 그 값들이 차례차례 다 나뉘어 들어가는 것이기 때문에 이런 결과가 나온다.

a = ['hi', 'hello', '안녕', '하세요']
a.extend(['반갑습니다'])
print(a)
# ['hi', 'hello', '안녕', '하세요', '반갑습니다'] 이 프린트된다. extend 를 사용하여 문자열 그대로 리스트 안에 넣고 싶으면 문자열을 리스트로 묶으면 된다.
```



## 3. 복사가 잘 된 건가?

```python
a = [1, 2, 3, 4, 5]
b = a

a[2] = 5

print(a)
print(b)

# 둘 다 [1.2.5.4.5] 이 프린트된다. b = a 라는 것은 b 가 a object를 끌어다가 쓴다는 것이기 때문에 같은 곳을 바라보고 있기 때문이다. 즉, 같은 object 을 참조하는 것이기 때문에 a[2] = 5 에서 object 의 변화가 생겼으므로 a, b 모두 변화가 생기게 된다.

# 따라서 프린트 된 list 요소가 같다.
```
